## 새 PC에서 앱을 설치한 경우

머신 설정은 **PC 로컬**에 저장됩니다. 새 컴퓨터에서는 아래를 다시 확인하세요.

- [ ] 동일 계정으로 로그인
- [ ] USB 드라이버 / Phidget Network Server 재설치
- [ ] 머신 추가·시리얼 포트·IP·슬레이브 ID 재입력
- [ ] 로스터 용량 등 머신 메타데이터 입력

## USB 시리얼 로스터

1. **설정 → 로스팅 머신**에서 해당 머신 선택 → **연결** 시도
2. **시리얼 포트**가 올바른지 확인 (USB 재연결 시 COM 번호가 바뀔 수 있음)
3. 보드레이트·슬레이브 ID·레지스터 주소가 매뉴얼과 일치하는지 확인
4. 포트 목록이 비면 → [시리얼 포트 드라이버](/machines/serial-driver/)
5. 드라이버 설치 후에도 실패하면 **다른 PC**에서 테스트
6. 다른 PC에서도 실패하면 USB 케이블·허브·로스터 본체 점검 → 제조사 AS

## LAN (Modbus TCP) 로스터

1. 로스터와 PC가 **같은 네트워크**인지 확인
2. 로스터 **IP 변경** 여부 확인 — 변경 시 앱 설정 IP 수정
3. `ping <로스터 IP>` 로 네트워크 도달 여부 확인 (사내 정책에 따라 불가할 수 있음)
4. 공유기·방화벽에서 **502 포트** 차단 여부 확인
5. Giesen·Probat 등은 제조사 네트워크 설정 선행 필요
6. 다른 PC에서도 실패하면 로스터·스위치·케이블 점검

## Phidget

1. **Phidget Control Panel** → Network Server **Running** 확인
2. Devices 탭에서 온도 갱신 확인
3. First Crack 채널 매핑이 Control Panel과 일치하는지 확인
4. → [Phidget 연동](/machines/phidget/) 재설정

## 온도는 나오는데 값이 이상할 때

| 현상 | 점검 |
|------|------|
| 항상 0 | 잘못된 레지스터 주소·슬레이브 ID |
| 10배/10분의 1 | 스케일 팩터(÷10, ÷100) 조정 |
| BT·ET 뒤바뀜 | 채널 매핑 스왑 |
| 끊김·튐 | USB 케이블, RS485 종단 저항, 전원 노이즈 |

Artisan에서 정상이었다면 [Artisan 설정](/machines/artisan/)과 값을 1:1로 비교하세요.

## 앱·동기화 문제

- **로그인 실패:** 웹에서 동일 계정으로 로그인되는지 확인
- **프로파일이 안 보임:** 인터넷 연결 후 동기화 대기 또는 앱 재시작
- **구독 인식 안 됨:** [계정 페이지](https://firstcrackiscoming.com/account)에서 Pro 상태 확인
