## 新しい PC にアプリを入れた場合

マシン設定は **その PC のローカル** に保存されます。新しいコンピュータでは次をやり直してください。

- [ ] 同じアカウントでログイン
- [ ] USB ドライバ / Phidget Network Server の再インストール
- [ ] マシン追加・シリアルポート・IP・スレーブ ID の再入力
- [ ] バッチ容量などのメタデータ再入力

## USB シリアルロースター

1. **設定 → 焙煎機** で対象マシンを選び **接続**
2. **シリアルポート** が正しいか（再接続で COM が変わることがあります）
3. ボーレート・スレーブ ID・レジスタがマニュアルと一致するか
4. ポート一覧が空 → [シリアルポートドライバー](/ja/machines/serial-driver/)
5. それでも失敗 → **別 PC** でテスト
6. 別 PC でも失敗 → ケーブル・ハブ・本体点検 → メーカー AS

## LAN（Modbus TCP）

1. ロースターと PC が **同じネットワーク** か
2. ロースター **IP 変更** の有無 — 変更時はアプリも更新
3. ポリシーが許せば `ping <ロースター IP>`
4. ルータ・ファイアウォールで **502** が塞がれていないか
5. Giesen・Probat などはメーカーのネットワーク設定が先
6. 他 PC でも失敗 → ロースター・スイッチ・ケーブル

## Phidget

1. **Phidget Control Panel** → Network Server が **Running**
2. Devices で温度が更新されるか
3. アプリのチャンネル対応が Control Panel と一致するか
4. → [Phidget 連携](/ja/machines/phidget/) を再設定

## 温度は出るが値がおかしい

| 現象 | 確認 |
|------|------|
| 常に 0 | レジスタアドレス・スレーブ ID の誤り |
| 10 倍 / 1/10 | スケール（÷10、÷100） |
| BT・ET が逆 | チャンネルマッピングの入れ替え |
| 途切れ・ノイズ | USB ケーブル、RS485 終端、電源ノイズ |

Artisan では正常なら [Artisan 設定](/ja/machines/artisan/) と 1:1 で比較してください。

## アプリ・同期

- **ログイン失敗:** Web でも同じアカウントで入れるか
- **プロファイルが見えない:** ネット接続後の同期待ち、または再起動
- **サブスク未認識:** [アカウントページ](https://firstcrackiscoming.com/account) で Pro 状態を確認

## それでも解決しないとき

[お問い合わせ](/ja/support/contact/) に次を含めてください。

- OS・アプリバージョン
- ロースターメーカー・モデル
- 接続方式（USB / LAN / Phidget）
- エラーメッセージのスクショ
- Artisan 設定キャプチャ（あれば）
