## 接続できないとき

1. **プラン** — 注文メニューがロックなら **Pro+** または **Enterprise**（注文連携）か確認
2. **コピー** — Client ID / Secret / token の前後空白・改行を除去
3. **プラットフォーム側権限**
   - Naver: SELF アプリ、有効な Secret
   - Shopify: アプリインストール済み、`read_orders`
   - Cafe24: `mall.read_order`、トークン未期限切れ
4. チャンネルカードの **last error** を確認
5. キーを再発行したら **切断して再接続**

## 注文が入らないとき

| 確認 | 説明 |
|------|------|
| 同期実行 | **今すぐ同期** を押したか、最終同期時刻 |
| 期間 | Shopify 約 60 日、Cafe24 の start/end |
| ステータス | 未払い・テスト注文のみでないか |
| プラットフォーム管理画面 | 実注文が管理画面にあるか |

Naver は **変更された商品注文** 基準で収集します。古い未変更注文は次のステータス変更後に拾われることがあります。

## 焙煎必要量が 0 のとき

1. ステータスが **paid / preparing** でまだ **fulfilled** でないか
2. 明細が **active** か（キャンセル・返金行を除く）
3. [SKU マッピング](/ja/orders/sku-mapping/) 済みか（`product_id`）
4. 商品に生豆/ブレンド・正味重量があるか

## セキュリティ

- First Crack スタッフに **Client Secret / access token 全文を送らないでください**。
- 漏えいが疑わしい場合はプラットフォームでキー・トークンを **即時失効・再発行** し、First Crack チャンネルを再接続してください。

## 問い合わせ時に含めるとよい情報

- プラットフォーム（Naver / Shopify / Cafe24）
- ショップ・Mall ID または表示名（シークレット除く）
- おおよその時刻
- 画面上のエラー文
- 同期は成功だが特定注文だけ無いか
