---
title: 商品 SKU マッピング
description: EC 商品と First Crack の焙煎商品を紐づける
---

注文が入っても **SKU が一致しない** と焙煎必要量・自動出荷の対象から外れます。  
同期時、First Crack は次の規則でローカル商品を紐づけます。

## マッピング規則

注文明細のプラットフォーム ID のいずれかが **`products.sku` と文字列一致** すると接続されます。

| プラットフォーム | 優先マッチ項目 |
|------------------|----------------|
| Naver | `productId`、`optionCode` / `optionManageCode`、（可能なら）販売者商品コード |
| Shopify | product id、**variant id**、variant **SKU** |
| Cafe24 | `product_no`、`variant_code`、`product_code` |

**いちばん簡単な方法:** EC に登録した販売者管理コード / SKU を First Crack 商品の **SKU** に **そのまま** 入れます。

## 設定手順

1. First Crack **注文** → **商品** で販売 SKU を作成（名前、正味 g、生豆/ブレンド、プロファイル、**SKU**）
2. EC 側の販売者コード・SKU に同じ文字列を入れる（既にある場合は First Crack 側を合わせる）
3. **チャンネル** で **今すぐ同期**
4. **注文** タブで明細に商品が付いたか確認
5. ダッシュボード・注文画面の **焙煎必要量** に反映されるか確認

## 例

| First Crack 商品 | SKU | EC |
|------------------|-----|-----|
| エチオピア 200g | `ETH-200` | Naver 販売者商品コード `ETH-200` |
| ハウスブレンド 1kg | `HB-1KG` | Shopify variant SKU `HB-1KG` |
| デカフェ 250g | `DEC-250` | Cafe24 商品コード `DEC-250` |

## 未マッピングのとき

- 注文一覧には出るが **焙煎必要量に載らない**
- 出荷時にその明細の在庫減がスキップされることがある

SKU 修正後に **再同期** すると、以降の収集・更新でマッピングが付きます（保存済み明細は同期が更新するときに反映）。

## 手動注文

チャンネルなしで **注文 → 新規注文** からも入力できます。商品を選べば同じ必要量・出荷パイプラインを通ります。
