## 연결이 안 될 때

1. **플랜** — 주문 메뉴가 잠겨 있으면 **Pro+** 또는 **Enterprise**(주문 연동)인지 확인합니다.
2. **값 복사** — Client ID / Secret / token 앞뒤 공백, 줄바꿈 제거
3. **플랫폼 쪽 권한**
   - 네이버: SELF 앱, 시크릿 유효
   - Shopify: 앱 설치됨, `read_orders`
   - Cafe24: `mall.read_order`, 토큰 미만료
4. 채널 카드의 **last error** 문구를 확인합니다.
5. 키를 재발급했다면 **연결 해제 후 다시 연결**합니다.

## 주문이 안 들어올 때

| 확인 | 설명 |
|------|------|
| 동기화 실행 | **지금 동기화**를 눌렀는지, 마지막 동기화 시각 |
| 기간 | Shopify 60일 제한, Cafe24 start/end 기간 |
| 상태 | 결제 전·테스트 주문만 있는지 |
| 플랫폼 admin | 실제 주문이 플랫폼 관리자에 보이는지 |

네이버는 **변경된 상품주문** 기준으로 수집합니다. 아주 오래된 미변경 주문은 다음 상태 변경 후에 잡힐 수 있습니다.

## 로스팅 필요량이 0일 때

1. 주문 상태가 **paid / preparing** 이고 아직 **출고(fulfilled)** 되지 않았는지
2. 라인이 **active** 인지 (취소·환불 라인 제외)
3. [SKU 매핑](/orders/sku-mapping/) 되었는지 (`product_id` 연결)
4. 상품에 생두/블렌드·순중량이 설정되어 있는지

## 보안

- First Crack 직원에게 **Client Secret / access token 전문을 보내지 마세요.**
- 유출 의심 시 플랫폼에서 키·토큰을 **즉시 폐기·재발급**하고 First Crack 채널을 다시 연결하세요.

## 문의 시 포함하면 좋은 정보

- 플랫폼 (네이버 / Shopify / Cafe24)
- 샵·Mall ID 또는 스토어 표시 이름 (시크릿 제외)
- 대략적인 시각
- 화면에 보이는 에러 문구
- 동기화는 성공인데 특정 주문만 없는지
