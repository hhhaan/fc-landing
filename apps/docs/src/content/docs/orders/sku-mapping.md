---
title: 상품 SKU 매핑
description: 쇼핑몰 상품과 First Crack 로스팅 상품을 연결하는 방법
---

주문이 들어와도 **SKU가 맞지 않으면** 로스팅 필요량·자동 출고 대상에서 빠집니다.  
First Crack은 동기화 시 아래 규칙으로 로컬 상품을 붙입니다.

## 매핑 규칙

주문 라인의 플랫폼 ID 중 하나가 **`products.sku`와 문자열 일치**하면 연결됩니다.

| 플랫폼 | 우선 매칭에 쓰는 값 |
|--------|---------------------|
| 네이버 | `productId`(채널 상품번호), `optionCode` / `optionManageCode`, (가능 시) 판매자 상품 코드 |
| Shopify | product id, **variant id**, variant **SKU** |
| Cafe24 | `product_no`, `variant_code`, `product_code` |

**가장 간단한 방법:** 쇼핑몰에 등록한 판매자 관리 코드 / SKU 를 First Crack 상품 **SKU** 필드에 **그대로** 넣습니다.

## 설정 순서

1. First Crack **주문** → **상품** 탭에서 판매 SKU를 만듭니다.
   - 이름, 순중량(g), 생두/블렌드, 로스팅 프로파일, **SKU**
2. 쇼핑몰 상품 관리 화면에서 같은 문자열을 판매자 코드·SKU에 넣습니다. (이미 있으면 First Crack SKU를 쇼핑몰에 맞춤)
3. **채널**에서 **지금 동기화**
4. **주문** 탭에서 라인에 상품이 붙었는지 확인합니다.
5. 대시보드·주문 화면의 **로스팅 필요량**에 반영되는지 확인합니다.

## 예시

| First Crack 상품 | SKU | 쇼핑몰 |
|------------------|-----|--------|
| 에티오피아 200g | `ETH-200` | 네이버 판매자 상품 코드 `ETH-200` |
| 하우스 블렌드 1kg | `HB-1KG` | Shopify variant SKU `HB-1KG` |
| 디카페인 250g | `DEC-250` | Cafe24 상품코드 `DEC-250` |

## 미매핑일 때

- 주문은 목록에 보이지만 **로스팅 필요량에 안 잡힘**
- 출고 시 해당 라인 재고 차감이 스킵될 수 있음

SKU를 수정한 뒤 **다시 동기화**하면 이후 수집·업데이트 시 매핑이 붙습니다. (이미 저장된 라인은 동기화가 라인을 갱신할 때 반영)

## 수동 주문

채널 없이 **주문 → 새 주문**으로 수동 입력할 수 있습니다. 이때도 상품을 고르면 동일하게 필요량·출고 파이프라인을 타게 됩니다.
